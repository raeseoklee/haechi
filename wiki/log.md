## [2026-06-16] fix | SSRF IPv4-mapped IPv6 normalization — P1-CR-002 + P2-CR-012 (toward 1.3.1)

Second remediation PR for the 2026-06-16 register (G9). **P1-CR-002 (confirmed real):** `isBlockedAddress` blocked the *dotted* IPv4-mapped IPv6 form (`::ffff:127.0.0.1`) but classified the *hex* form (`::ffff:7f00:1` = 127.0.0.1, `::ffff:a00:1` = 10.0.0.1) as PUBLIC → an SSRF bypass to loopback/RFC1918. Root cause: each copy only handled `isIP(slice)===4` (dotted). Fix: added pure helpers `ipv6ToBytes()` + `mappedIpv4()` to each copy — parse the literal to 16 octets, and treat the last 32 bits as an embedded IPv4 *only* when bytes 0..9 are zero AND bytes 10..11 are `0xffff` (the `::ffff:0:0/96` prefix), then run the v4 private/loopback/link-local/metadata check on the embedded quad.
