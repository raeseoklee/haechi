## Principles behind the ordering

1. **Product credibility before operational controls** â€” for a personal/self-hosted preview, token round-trip and MCP plug-in UX persuade more than rate limits.
2. **First publish small** â€” name ownership decoupled from feature risk; the provenance/trusted-publishing pipeline remains an open pre-0.4-release task.
3. **Injection heuristics ship report-only** (default action `allow`; detections still audit) â€” false-positive blocks would burn trust in a security product.
4. **No dynamic provider loading before the sandbox** â€” P1-SEC-004's manifest-only stance held through 0.9; 1.0 lifts it **narrowly** (signed `authProvider`-only, isolated, audited â€” [[plugin-sandbox]]), recorded as P1-SEC-024 superseding it. Injection stays the default for every other provider.
