| 1.1.0 | Plugin capability **enforcement** | **Design pinned 2026-06-11** (`docs/current/release-1.1-implementation-scope.md`, hardened after a 3-lens adversarial review with **empirical Node-26 testing** — three critical first-draft errors fixed). Closes the most-cited **1.0 honest residual**: `worker_threads` ≠ capability sandbox, so a malicious *signed* plugin could still use fs/net + exfiltrate its credential. Adds an opt-in **`process-isolated`** runtime ([[plugin-sandbox]]) — `createProcessIsolatedAuthProvider` runs a signed `authProvider` in a child `node` under `--permission` with **zero grants**, loaded from a `data:` URL (no fs grant → no TOCTOU/symlink surface), `stdio:['ignore','ignore','ignore','ipc']` (closes the stdout/stderr/fd leak channel `--allow-net` can't gate), scrubbed env, JSON-string-only IPC. Network containment is the **kernel-enforced `--allow-net` denial** — NOT a "delete `node:net`" harness (`process.binding('tcp_wrap')` + a fresh `import('node:net')` bypass that trivially); default **`netEnforcement:"require-permission"` fails closed** (throws) on a Node without `--allow-net` (Node 22 LTS) rather than pretending to contain. Credentials: the **host** runs `createJwtVerifier` for standard JWT (no plugin sees it); `process-isolated` is for **custom/opaque** creds, contained by net+stdio+fs denial; key material is **host-fetched from operator-declared URLs** (no plugin-driven SSRF) — **promoting `isBlockedAddress`/`guardedFetch` into a core node:-only module** (core stays zero-dep; satellites re-import, ending the drift). Lifecycle audit gains **host-computed/enum-only** fields, outside the frozen protect-event schema. `worker-isolated` (1.0) stays unchanged. Risks **P1-SEC-026** (process-isolated enforcement, strengthens P1-SEC-024) / **P1-SEC-027** (host-mediated key material + core SSRF guard). 3-PR stack. ([[packaging-and-distribution]], [[identity-and-auth]]) |

## Principles behind the ordering

1. **Product credibility before operational controls** — for a personal/self-hosted preview, token round-trip and MCP plug-in UX persuade more than rate limits.
2. **First publish small** — name ownership decoupled from feature risk; the provenance/trusted-publishing pipeline remains an open pre-0.4-release task.
3. **Injection heuristics ship report-only** (default action `allow`; detections still audit) — false-positive blocks would burn trust in a security product.
4. **No dynamic provider loading before the sandbox** — P1-SEC-004's manifest-only stance held through 0.9; 1.0 lifts it **narrowly** (signed `authProvider`-only, isolated, audited — [[plugin-sandbox]]), recorded as P1-SEC-024 superseding it. Injection stays the default for every other provider.
